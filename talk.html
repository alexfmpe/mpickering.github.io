Generating this map

  • Built around a free monad.
  • We create a data structure which describes which keywords each AST element could have.
class Data ast => Annotate ast where
  markAST :: GHC.SrcSpan -> ast -> Annotated ()
data instance T Int = A

    mark GHC.AnnType

Monad-Fail Proposal

  • Removing the fail method from the monad typeclass
  • Simple, well-defined refactoring.
  • A tool which library authors can use to update and review the changes.
{-# LANGUAGE Haskell2010 #-}

module Maybe1 where

data Maybe' a = Nothing' | Just' a

instance Monad Maybe' where
    return = Just'

    Nothing' >>= _ = Nothing'
    Just' x  >>= f = f x

    fail _ = Nothing'
module Maybe1 where

data Maybe' a = Nothing' | Just' a

instance Functor Maybe' where
    fmap f m    = m >>= pure . f

instance Applicative Maybe' where
    pure = Just'
    f1 <*> f2   = f1 >>= \v1 -> f2 >>= (pure . v1)

instance Monad Maybe' where
    Nothing' >>= _  = Nothing'
    Just' x  >>= f  = f x

instance MonadFail Maybe' where
    fail _ = Nothing'

Extending GHC: Making Pattern Synonyms more like data constructors

  • Problem: You currently have to explicitly import and export pattern synonyms.
  • Special keyword as they are in the same namespace as data constructors.
{-# LANGUAGE PatternSynonyms #-}

module Foo(pattern P) where

data A = A

pattern P :: A
pattern P = A
{-# LANGUAGE PatternSynonyms #-}
module Baz where

import Foo(pattern P)
  • The consumer is aware they are using a pattern synonym, no good!
  • Solution: Associate pattern synonyms with type constructors when exporting
{-# LANGUAGE PatternSynonyms #-}

-- P is *not* a data constructor for A
module Foo(A(P)) where

data A = A

pattern P :: A
pattern P = A
{-# LANGUAGE PatternSynonyms #-}

module Baz where

-- Also imports `P`.
import Foo (A(..))
  • The user is no longer aware whether P is a pattern synonym or an actual constructor for A. Good!

Record Pattern Synonyms

  • We can define prefix and infix pattern synonyms but not record pattern synonyms.
  • Pattern synonyms which act just like record data constructors.
{-# LANGUAGE PatternSynonyms #-}

module Foo where

data Point = Point (Int, Int)

pattern MyPoint :: Int -> Int -> Point
pattern MyPoint {x, y} = Point (x,y)
data Point = MyPoint { x :: Int, y :: Int }
